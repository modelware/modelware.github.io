Everything the platform can do interactively, it can also do unattended. The OML Code CLI is a production-grade command line that brings the full power of the platform to scripts, terminals, and pipelines.

Lint your models for style and structure. Validate them against your methodology. Run the reasoner to derive entailments and check consistency. Execute queries and analyses. Render viewpoints, reports, and gate products. Each operation is a composable command that behaves the same on a laptop and on a build server.

That consistency is what makes model DevOps real. The same checks an engineer runs locally before committing are the checks the CI pipeline runs on every commit, so nothing inconsistent ever lands in the baseline. And because the CLI is scriptable, your team can automate the repetitive work that consumes engineering hours today.

#### What you get

- One command line for lint, validate, reason, query, and render
- Identical behavior locally and in CI, so results are reproducible everywhere
- The building block of commit gates, scheduled analyses, and release packaging
- Clean integration with Git workflows, build systems, and artifact repositories

The most convincing thing we can show a skeptical engineer is a pipeline that catches a model error automatically. Ask for that demo.
